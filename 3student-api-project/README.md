# Руководство по использованию API системы управления студентами

### Использование с Docker

```bash
# 1. Запуск всех сервисов
docker-compose up --build -d

# 2. Доступ к документации API
# Откройте в браузере: http://localhost:8000/docs
```

### Локальный запуск

```bash
# 1. Активация виртуального окружения
venv\Scripts\activate

# 2. Установка зависимостей
pip install -r requirements.txt

# 3. Запуск приложения
python main.py

# 4. Доступ к документации API
# Откройте в браузере: http://localhost:8000/docs
```

## Адреса документации API

| Адрес | Описание |
|------|------|
| http://localhost:8000/docs | Интерактивная документация API (рекомендуется) |
| http://localhost:8000/redoc | Альтернативная документация API |
| http://localhost:8000/openapi.json | Файл спецификации OpenAPI |

## Доступные конечные точки API

### Базовая информация
- **GET /** — базовая информация об API  
- **GET /health** — проверка работоспособности (health check)

### Управление студентами
| Метод | Конечная точка | Описание |
|------|----------------|----------|
| GET | `/students/` | Получить список всех студентов |
| POST | `/students/` | Создать нового студента |
| GET | `/students/{id}` | Получить информацию о конкретном студенте |
| DELETE | `/students/{id}` | Удалить студента |

### Управление группами
| Метод | Конечная точка | Описание |
|------|----------------|----------|
| GET | `/groups/` | Получить список всех групп |
| POST | `/groups/` | Создать новую группу |
| GET | `/groups/{id}` | Получить информацию о конкретной группе |
| DELETE | `/groups/{id}` | Удалить группу |

### Управление связями «студент–группа»
| Метод | Конечная точка | Описание |
|------|----------------|----------|
| POST | `/students/{student_id}/groups/{group_id}` | Добавить студента в группу |
| DELETE | `/students/{student_id}/groups/{group_id}` | Удалить студента из группы |
| GET | `/groups/{group_id}/students` | Получить всех студентов в указанной группе |
| PUT | `/students/{student_id}/transfer` | Перевести студента в другую группу |

## Примеры форматов данных

### Объект студента
```json
{
  "id": 1,
  "first_name": "Сань Чжан",
  "last_name": "Чжан",
  "email": "zhangsan@example.com"
}
```

### Объект группы
```json
{
  "id": 1,
  "name": "Компьютерные науки",
  "code": "CS101",
  "description": "Группа по специальности «Компьютерные науки»"
}
```

2. Доступ к документации: http://localhost:8000/docs  
3. Проверка состояния контейнеров: `docker-compose ps`
